# **图像画质分析工具**

一款跨平台的移动端图像画质评估工具，支持本地相册/文件加载、预览图片，并基于传统计算机视觉算法自动给出清晰度、曝光、噪点三个维度的评分，最终输出0-100综合画质评分及可读评语。

## 功能特点

多格式支持：JPEG/JPG/PNG/BMP等常见格式
大图安全加载：自动缩放至1200px分析，避免内存溢出，支持≥12MP原图
三维度评分：清晰度（拉普拉斯方差）、曝光（过曝/欠曝比例）、噪点（中值滤波残差法）
综合评分：加权(4:3:3)融合三个维度，输出0-100分及文字描述
跨平台：桌面端(Windows/Linux)可直接运行，移动端可通过Buildozer打包为Android APK

## 桌面端运行

**环境要求**
Python 3.9+
**安装依赖**
pip install kivy opencv-python numpy pillow
**运行**
python main.py

使用[Buildozer]在WSL或Linux环境下打包：
buildozer init
buildozer android debug
生成的APK位于bin/目录，传输至手机安装即可。

## 使用说明

1.点击“选择图片”从本地文件系统中选取图片（支持JPG/PNG/BMP）。
2.图片加载后会在预览区显示缩略图，并展示原始分辨率。
3.点击“开始分析”，程序将自动计算并在下方显示：
综合画质评分（0-100）
各维度子分与详细数值
可读评价（如“非常清晰，曝光正常，噪点极少”）
4.所有分析均在缩略图（长边≤1200px）上完成，保证性能。

## 评分算法说明

**1.清晰度：拉普拉斯方差(Laplacian Variance)**
原理：利用3×3拉普拉斯算子检测图像二阶导数（边缘强度），计算全图方差。图像越清晰，边缘越锐利，方差越大。
来源：Pech-Pacheco,J. L. et al.(2000).Diatom autofocusing in brightfield microscopy.
归一化：sharp_norm = min(1.0,raw / 700.0)
噪点惩罚：sharp_norm *= 0.7 + 0.3 × noise_norm，避免高噪点伪装成细节。
局限性：对纹理内容敏感：平坦区域（天空、墙壁）即使清晰方差也偏低;大尺寸图像统一缩放后分析，分辨率影响已消除，但极端纹理差异仍存在。
**2.曝光：直方图过曝/欠曝比例+平均亮度**
原理：统计灰度直方图中过曝（亮度>240）和欠曝（亮度<15）的像素占比，结合平均亮度是否落在舒适区(70-180)给予奖励。
公式：
base = 1.0 - (over_ratio + under_ratio)
bonus = 0.1 if 70 < mean < 180 else 0
score = clamp(base + bonus, 0, 1)
参数：过曝阈值240，欠曝阈值15，理想亮度区间70-180（经验值）。
局限性：全局统计不考虑主体位置，高调/低调艺术摄影可能被误判为过曝/欠曝；阈值固定，对不同场景的适应能力有限。
**3.噪点：中值滤波残差法(Median Filter Residual)**
原理：对灰度图像做3×3中值滤波，计算原图与滤波后图像的差值方差，并归一化。
来源：Immerkaer,J.(1996). Fast Noise Variance Estimation, CVIU.
归一化：score = max(0.1, 1.0 - noise_var × 1.2)
局限性：难以区分真实纹理（如织物、草地）与随机噪点，会导致纹理丰富区域噪点得分偏低；对彩色噪点（三通道独立处理）检测能力有限，可能漏检；受JPEG块效应影响，压缩严重时噪声方差可能异常。
**综合评分**
Total = 0.4×清晰度 + 0.3×曝光 + 0.3×噪点，四舍五入映射至0-100。

## 各图像格式差异与解码注意事项

| 格式      | 压缩方式      | 位深     | 透明 | 注意事项                                             |
| --------- | ------------- | -------- | ---- | ---------------------------------------------------- |
| JPEG(JPG) | 有损(DCT)     | 8 bit    | 无   | 多次压缩引入块效应，影响噪点估计；OpenCV忽略EXIF方向 |
| PNG       | 无损(DEFLATE) | 8/16 bit | 支持 | 解码较慢，大图需注意内存；无压缩噪声，噪点分析更准确 |
| BMP       | 无压缩        | 8 bit    | 无   | 体积大；OpenCV完美支持                               |
